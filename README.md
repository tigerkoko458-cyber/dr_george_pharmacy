# نظام محاسبة صيدلية د/ جورج مجدي

تطبيق Flutter + SQLite لإدارة الحسابات اليومية للصيدلية (بدون إدارة أصناف/مخزون).

## المميزات المطبّقة

- تسجيل دخول وصلاحيات مستخدمين (مدير / مسؤول / موظف) قابلة للتخصيص لكل مستخدم
- إدارة المبيعات على 3 شيفتات يوميًا (نقدي / فيزا / شركات / أخرى) مع إقفال كل شيفت
- إدارة الشركات بكود ثابت لا يتغير + نسبة خصم لكل شركة
- تسجيل فواتير الشركات يوميًا وتجميعها في **مطالبة شهرية واحدة فقط لكل شركة**
- التحصيلات من الشركات مع ربطها تلقائيًا بالمطالبة وتحديث حالتها (مفتوحة / مُرسلة / تحصيل جزئي / محصلة بالكامل)
- الخزينة: كل حركة نقدية (مبيعات شيفت، تحصيل، مصروف) تُرحّل تلقائيًا وتُحسب الرصيد لحظيًا
- تقرير المبالغ غير المحصلة لكل الشركات
- المصروفات مع خصمها تلقائيًا من رصيد الخزينة
- تقارير مالية بفترة زمنية مع تصدير PDF و Excel
- إقفال اليوم (يتطلب إقفال الشيفتات الثلاثة أولًا) مع تقرير PDF
- نسخ احتياطي/استرجاع لقاعدة البيانات بالكامل (نسخ ومشاركة ملف .db)
- دعم كامل للغة العربية واتجاه الكتابة RTL، وشعار الصيدلية مستخدم في شاشة الدخول والصفحة الرئيسية والتقارير

## خطوات التشغيل

هذا المجلد يحتوي فقط على مجلد `lib/` وملف `pubspec.yaml` والأصول (الشعار) — بدون
مجلدات المنصات (android/ios) لأن بيئة الإنشاء هنا لا تملك Flutter SDK ولا اتصال
إنترنت لتوليدها واختبارها. لتشغيل المشروع فعليًا على جهازك:

1. تأكد من تثبيت Flutter SDK (قناة stable) على جهازك.
2. أنشئ مشروع Flutter فارغ بنفس الاسم ثم انسخ هذه الملفات فوقه:
   ```bash
   flutter create dr_george_pharmacy
   cd dr_george_pharmacy
   # انسخ محتوى مجلد lib/ و pubspec.yaml و assets/ من هذا الأرشيف
   # إلى المجلد الجديد (استبدال الملفات الموجودة)
   ```
3. نزّل الحزم:
   ```bash
   flutter pub get
   ```
4. شغّل التطبيق:
   ```bash
   flutter run
   ```

### ملاحظة عن خطوط PDF

تصدير PDF يستخدم خط "Noto Naskh Arabic" عبر مكتبة `printing` (PdfGoogleFonts)
والذي يُحمَّل تلقائيًا من الإنترنت أول مرة فقط، ثم يُخزَّن محليًا. تأكد من
توفر اتصال إنترنت عند أول عملية طباعة/تصدير PDF.

### الصلاحيات المطلوبة على أندرويد

بعد تشغيل `flutter create`، تأكد أن ملف `android/app/src/main/AndroidManifest.xml`
يحتوي على صلاحية الإنترنت (لتحميل الخطوط) وهي مفعّلة افتراضيًا في مشاريع Flutter.

## هيكل المشروع

```
lib/
  db/db_helper.dart          - إنشاء قاعدة البيانات وجداولها بالكامل
  models/                    - نماذج البيانات (User, Company, Shift, ...)
  providers/                 - منطق الأعمال وإدارة الحالة (Provider)
  screens/                   - كل شاشات التطبيق
  utils/                     - أدوات مساعدة (تشفير، PDF، Excel، نسخ احتياطي، تنسيق، الثيم)
  main.dart                  - نقطة تشغيل التطبيق وربط كل الـ Providers
assets/images/logo.png       - شعار الصيدلية
```

## نقاط تحتاج مراجعتك قبل الاستخدام الفعلي

- تم بناء الكود بالكامل يدويًا دون تشغيله فعليًا على Flutter SDK حقيقي (البيئة
  التي أُنشئ فيها لا تملك Flutter ولا اتصال إنترنت)، لذا يُنصح بمراجعة أي أخطاء
  compile بسيطة قد تظهر عند أول `flutter pub get` / `flutter run` (مثل توافق
  إصدارات الحزم) وإخباري بها لتصحيحها فورًا.
- شاشة "مبيعات شركات" داخل الشيفت هي مبلغ إجمالي فقط للتسوية؛ التفاصيل الفعلية
  (فاتورة لكل شركة) تُسجَّل من شاشة "المطالبات الشهرية" حتى تتجمّع في المطالبة
  الشهرية الصحيحة لكل شركة.
